# AGENTS.md

## 목적
이 파일은 이 저장소에서 작업하는 기여자와 코딩 에이전트를 위한 공통 코딩 규칙을 정의합니다.

## 적용 범위 및 에이전트 사용
- 이 파일은 주로 이 저장소에서 동작하는 Codex 에이전트를 대상으로 합니다.
- 다른 에이전트(예: Gemini, Claude)는 이 파일을 자동으로 읽지 못할 수 있습니다.
- 도구가 저장소 지침을 자동 로드하지 않으면, 이 파일의 핵심 규칙을 해당 도구의 사용자 지정/프로젝트 지침에 복사해 사용하세요.
- 필요하다고 생각되면 알아서 Commit를 수행 합니다.

## 공통 코딩 규칙
- 기교보다 명확성을 우선합니다.
- 변경은 최소화하고 요청된 결과에 집중합니다.
- 요청 범위를 벗어나는 동작 변경은 하지 않습니다.
- 리팩터링을 명시적으로 요청받지 않았다면 기존 아키텍처와 네이밍 규칙을 유지합니다.
- 함수, 변수, 컴포넌트 이름은 의미가 분명하게 작성합니다.
- 중복 로직은 피하고, 실용적인 범위에서 공통 로직을 추출합니다.
- 코드만으로 의도가 명확하지 않을 때는 한국어로 주석을 추가합니다.

## 보안 규칙
- 비밀정보(API 키, 토큰, 비밀번호, 인증서)는 절대 커밋하지 않습니다.
- 환경 의존 자격증명이나 URL을 하드코딩하지 않습니다.
- 외부 입력을 검증하고 오류 경로를 명시적으로 처리합니다.
- 로그는 유용하게 남기되 민감한 데이터는 기록하지 않습니다.

## 프론트엔드 규칙
- 재설계를 요청받지 않았다면 기존 UI/UX 패턴을 유지합니다.
- 일반적인 모바일/데스크톱 화면 크기에서 반응형 동작을 보장합니다.
- 접근성(레이블, 포커스, 대비, 시맨틱)을 고려합니다.
- 작은 UI 변경에 불필요한 의존성 추가를 피합니다.

## API 및 데이터 규칙
- 가능하면 요청/응답 계약의 하위 호환성을 유지합니다.
- 네트워크 실패와 빈 상태를 안정적으로 처리합니다.
- 서버 응답은 방어적으로 파싱하고 검증합니다.

## 테스트 및 검증
- 가능하면 변경한 영역과 관련된 테스트/린트를 실행합니다.
- 테스트를 실행할 수 없으면 무엇을 검증하지 못했는지 문서화합니다.
- UI 변경 시 현실적인 수동 검증 시나리오를 최소 1개 포함합니다.

## Git 및 변경 위생
- 관련 없는 로컬 변경은 되돌리지 않습니다.
- 커밋은 의도가 명확한 원자 단위로 유지합니다.
- 동작 또는 설정이 변경되면 관련 문서도 함께 갱신합니다.

## 우선순위
규칙이 충돌하면 아래 순서를 따릅니다:
1. 채팅에서의 사용자 지시
2. 저장소 루트의 AGENTS.md
3. 로컬 모듈 관례
4. 개인 선호
